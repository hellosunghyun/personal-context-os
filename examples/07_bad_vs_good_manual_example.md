# 나쁜 예시와 좋은 예시

[← 이전](06_team_share_one_pager_example.md) | [목차](../SUMMARY.md) | [다음 →](08_source_register_example.md)

이 가이드는 문장을 더 멋있게 만드는 것이 아니라, 실제로 쓸 수 있게 만드는 것을 목표로 합니다.

## 너무 업그레이드된 예시

```markdown
이 사람은 본질을 꿰뚫는 전략가이며, 복잡한 문제 속에서도 시대적 방향성을 감각적으로 포착한다.
```

문제:

- 사용자가 실제로 말하지 않은 평가가 들어갔습니다.
- 공개용 자기소개처럼 멋있지만 근거가 약합니다.
- 팀원용 문서에 들어가면 과한 인상으로 읽힐 수 있습니다.

## 더 나은 예시

```markdown
이 사람은 문제를 보면 방향이 어긋나는 지점을 비교적 빨리 느끼는 편입니다.
다만 그 판단의 근거를 말로 설명하려면 과거 경험과 현재 조건을 정리할 시간이 필요합니다.
```

좋은 점:

- 실제 행동으로 설명합니다.
- 장점과 제약을 같이 말합니다.
- 협업자가 어떻게 도와야 할지 추론할 수 있습니다.

## 근거가 부족할 때

```markdown
추가 확인 필요:
- 이 판단이 실제로 반복된 패턴인지, 특정 상황에서만 나타난 반응인지 더 확인해야 합니다.
```
