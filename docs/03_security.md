# 보안과 경계

## 이 페이지에서 알게 되는 것

어떤 자료를 넣어도 되는지, 어떤 자료는 가명화해야 하는지, 어떤 자료는 넣지 말아야 하는지 정합니다.

---

## 먼저 결론

이 가이드는 개인 맥락을 다룹니다. 그래서 보안 기준이 필요합니다.

```text
내 이야기라도 회사와 타인의 정보가 섞이면 조심해야 한다.
민감한 원문은 그대로 넣지 않는다.
공개용, 팀원용, 본인용을 처음부터 나눈다.
```

---

## 넣어도 비교적 안전한 자료

```text
공개 블로그 글
공개 포트폴리오
공개 이력서
내가 직접 쓴 공개 가능한 자기소개
일하는 방식에 대한 일반 설명
```

---

## 가명화 후 사용할 자료

```text
회사 self-review 요약
인사평가 요약
동료 피드백 요약
프로젝트 회고
갈등 회고
면접 회고
```

### 처리 방법

```text
회사명 제거
실명 제거
고객명 제거
숫자는 범위로 바꾸기
내부 프로젝트명은 일반명으로 바꾸기
타인의 평가는 삭제하거나 행동 단위로 완화하기
```

---

## 본인용에만 조심해서 쓸 자료

```text
심리검사 결과
상담 메모
건강 관련 내용
가족/관계 이슈
강한 감정 기록
```

이런 자료는 협업자용이나 공개용에는 그대로 넣지 않습니다. 필요하다면 행동 패턴으로 바꿉니다.

```text
원문: 나는 불안이 심하다.
완화: 불확실한 상황에서 정보와 다음 행동이 없으면 빠르게 소진된다.
```

---

## 넣지 말아야 할 자료

```text
비밀번호
API Key
토큰
고객 데이터
계약서 원문
법무/분쟁 자료
타인의 개인정보
회사의 미공개 전략
접근 권한이 제한된 내부 문서 원문
```

---

## 공개 범위 표시법

모든 답변 옆에 공개 범위를 표시하세요.

```text
공개 가능
팀원용 가능
본인용만
비공개
추가 확인 필요
```
