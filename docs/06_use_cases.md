## 이 문서에서 다루는 내용

완성된 사용 설명서와 Active Context를 어떤 상황에 쓸 수 있는지 봅니다.

---

## 예시 1. 생각의 맥락을 설명하고 싶을 때

상황:

```text
어떤 의견은 직관적으로 떠오르는데, 왜 그렇게 생각하는지 말로 설명하기 어렵다.
```

사용 방식:

```text
Active Context를 첨부한다.
의견을 한 문장으로 적는다.
“제가 어떤 맥락에서 이런 생각을 하는지 설명 가능한 방식으로 정리해줘.”라고 요청한다.
```

예시 요청:

```text
첨부한 Active Context를 참고해줘.
저는 “이 프로젝트는 지금 자동화보다 문제 정의가 먼저다”라고 생각한다.
제가 어떤 맥락에서 이런 생각을 하는지, 팀원을 설득할 수 있는 방식으로 정리해줘.
단, 제가 말하지 않은 근거는 만들지 말고 가설은 가설로 표시해줘.
```

## 예시 2. 한 줄 아이디어를 점점 키울 때

사용 방식:

```text
1. 한 줄로 정리해달라고 한다.
2. 마음에 들면 10줄로 늘린다.
3. 또 마음에 들면 50줄로 늘린다.
4. 계속 원하는 결에 맞는지 확인하면서 키운다.
```

예시 요청:

```text
첨부한 Active Context를 참고해서 아래 아이디어를 제 말투와 판단 기준에 맞게 한 줄로 정리해줘.
마음에 들면 다음에 10줄, 50줄, 100줄로 확장할 예정이다.
너무 멋있게 만들지 말고 제가 쓴 것처럼 자연스럽게 써줘.

아이디어: 개인의 커리어 회고와 LLM 맥락 파일을 연결해서 답변 만족도를 높이는 워크북
```

## 예시 3. 팀원과 말이 막힐 때

상황:

```text
서로의 주장이 틀린 것은 아닌데 맥락이 어긋난다.
```

사용 방식:

```text
본인 설명서와 상대가 공유한 설명서를 함께 참고한다.
직접 판단을 맡기지 않는다.
대화 전에 가능한 오해 지점을 정리한다.
```

예시 요청:

```text
아래는 A와 B가 각자 공유한 협업용 사용 설명서 요약이다.
두 사람이 현재 의견이 갈리는 이유를 단정하지 말고, 가능한 맥락 차이로 정리해줘.
마지막에는 직접 대화에서 물어볼 질문 5개만 제안해줘.
```

실제 대화에서는 이렇게 쓸 수 있습니다.

```text
혹시 이건 당신이 이런 맥락에서 말한 걸까요?
제가 이해한 게 맞는지 확인하고 싶습니다.
```

## 예시 4. 피드백을 소화할 때

```text
첨부한 Active Context를 참고해줘.
아래 피드백을 사실, 감정, 추론, 다음 행동으로 나눠줘.
제가 방어적으로 받아들이기 쉬운 부분이 있으면 표시해줘.
단, 저를 위로하려고 좋은 말로 덮지 말고 실제로 바꿀 수 있는 행동만 정리해줘.
```

## 예시 5. 본인용과 공개용을 나눌 때

```text
아래 문단은 본인용으로는 맞지만 공개하기에는 민감할 수 있다.
첨부한 사용 설명서의 공개 기준을 참고해 공개용 표현으로 바꿔줘.
사적인 감정이나 건강 정보는 제거하고, 일하는 방식과 협업 조건으로 바꿔줘.
```
