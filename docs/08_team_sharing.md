# 팀원과 공유하기

[← 이전](07_limitations.md) | [목차](../SUMMARY.md) | [다음 →](09_gpts_and_bots.md)

## 이 페이지에서 알게 되는 것

완성된 사용 설명서를 팀원, 친구, 협업자와 어떻게 안전하게 공유할지 정합니다.

처음 사용 설명서를 만드는 중이라면 이 문서는 나중에 봐도 됩니다.

---

## 공유의 목적

목적은 상대를 분석하는 것이 아닙니다.

```text
서로를 덜 오해하기
피드백 방식을 맞추기
갈등 전에 맥락을 확인하기
업무 요청 방식을 개선하기
대화가 막혔을 때 질문을 더 잘 만들기
```

---

## 공유하면 좋은 내용

```text
일하는 방식
좋은 업무 요청 방식
피드백을 잘 받는 방식
싫어하는 소통 방식
집중이 잘 되는 조건
오해가 생기기 쉬운 지점
갈등이 생겼을 때 확인 질문
```

---

## 공유하지 않아도 되는 내용

```text
사적인 감정 기록
건강/심리 세부 정보
가족/관계 이슈
회사 내부 민감 정보
타인에 대한 평가
```

---

## 팀원과 쓸 때 좋은 질문

```text
제가 이렇게 이해했는데 맞나요?
이건 당신이 중요하게 생각하는 기준과 충돌한 걸까요?
지금 필요한 건 결론인가요, 맥락인가요?
제가 너무 해결책부터 말했나요?
이 대화에서 먼저 맞춰야 할 전제는 뭘까요?
```

---

## 나쁜 사용법

```text
설명서를 근거로 상대를 단정하기
상대에게 문서를 강요하기
민감한 내용을 팀에 공개하기
“이 사람은 원래 이래”라고 말하기
직접 대화 대신 GPT 판단을 들이밀기
```
