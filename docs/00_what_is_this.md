# 이 가이드는 무엇인가

[← 이전](../START_HERE.md) | [목차](../SUMMARY.md) | [다음 →](01_why_context_matters.md)

## 여기서 볼 것

이 가이드가 무엇인지, 무엇이 아닌지, 최종적으로 어떤 결과물을 만드는지 설명합니다.

---

## 한 문장 설명

이 문서는 개인 맥락을 모아, LLM이 **제가 납득할 수 있는 답변**을 더 자주 만들게 하려는 워크북입니다.

최신 도구 사용법을 다 따라가는 매뉴얼은 아닙니다. 제가 지금 실제로 쓰는 흐름을 정리했습니다. 아직 다른 사용 예와 비교해 크게 부족하다고 느끼지 않아서, 제품 기능이 바뀌어도 남겨야 할 것들을 파일로 남겨둡니다.

---

## 이 가이드가 해결하려는 문제

LLM 답변은 종종 이런 느낌을 줍니다.

```text
틀린 말은 아닌데 제 상황에는 애매하다.
일반론으로는 맞지만 제가 바로 쓰기 어렵다.
너무 멋있게 고쳐졌는데 제 말 같지 않다.
제가 왜 이런 생각을 하는지 설명해주지 못한다.
```

이 문제는 프롬프트를 더 화려하게 쓰는 것만으로 해결되지 않습니다.  
사람마다 좋은 답변을 판단하는 기준이 다르기 때문입니다.

---

## 여기서 말하는 사용 설명서란?

사람을 고정적으로 평가하는 문서가 아닙니다.

```text
본인을 이해하기 위한 맥락 문서
본인과 일하는 방법을 설명하는 협업 문서
LLM에게 답변 취향을 알려주는 참고 문서
시간이 지나면 계속 갱신되는 운영 문서
```

## 만들게 될 파일

처음에는 아래 세 파일이 핵심입니다.

```text
full_manual.md
- 길고 자세한 본인용 사용 설명서

active_context.md
- LLM에게 자주 첨부할 짧은 핵심 맥락

team_share_one_pager.md
- 팀원이나 협업자에게 공유할 수 있는 요약
```

필요하면 아래 공개용 파일도 만듭니다.

```text
public_profile.md
- 블로그, 포트폴리오, 자기소개에 쓸 수 있는 공개용 버전
```

## 이 가이드가 아닌 것

```text
성격검사 아님
심리 진단 아님
자기소개서 자동 생성기 아님
팀원을 분류하거나 라벨링하는 도구 아님
직접 대화를 대체하는 도구 아님
최신 AI agent 구축 방법론 아님
사람을 자동 평가하는 시스템 아님
```
