# 이 가이드는 무엇인가

[← 이전](00_why_now.md) | [목차](../SUMMARY.md) | [다음 →](01_why_context_matters.md)

## 이 문서에서 다루는 내용

이 가이드가 무엇인지, 무엇이 아닌지, 최종적으로 어떤 결과물을 만드는지 설명합니다.

---

## 한 문장 설명

이 문서는 개인 맥락을 모아, LLM이 **제가 납득할 수 있는 답변**을 더 자주 만들게 하려는 워크북입니다.

이 문서는 최신 도구 사용법을 따라가는 매뉴얼이 아닙니다. 도구가 바뀌어도 남는 문제를 다루는 개인 맥락 운영 문서입니다.

출처 메모: 이 문서의 중심은 특정 논문 하나가 아니라, LLM 제품 문서, context engineering, 개인정보 보호, 협업용 사용자 설명서 관행을 개인 워크북으로 엮어 만든 운영 방식입니다. 넓은 배경 자료는 [참고 자료](../references.md)에 모았습니다.

---

## 이 가이드가 해결하려는 문제

LLM 답변은 종종 이런 느낌을 줍니다.

```text
틀린 말은 아닌데 제 상황에는 애매하다.
일반론으로는 맞지만 제가 바로 쓰기 어렵다.
너무 멋있게 고쳐졌는데 제 말 같지 않다.
제가 왜 이런 생각을 하는지 설명해주지 못한다.
```

이 문제는 프롬프트를 더 화려하게 쓰는 것만으로 해결되지 않습니다.
사람마다 좋은 답변을 판단하는 기준이 다르기 때문입니다.

---

## 여기서 말하는 사용 설명서란?

사람을 고정적으로 평가하는 문서가 아닙니다.

```text
본인을 이해하기 위한 맥락 문서
본인과 일하는 방법을 설명하는 협업 문서
LLM에게 답변 취향을 알려주는 참고 문서
시간이 지나면 계속 갱신되는 운영 문서
```

---

## 만들게 될 문서

처음에는 아래 세 문서가 핵심입니다.

```text
전체 사용 설명서
- 길고 자세한 본인용 사용 설명서

Active Context
- LLM에게 자주 첨부할 짧은 핵심 맥락

팀원용 1페이지
- 팀원이나 협업자에게 공유할 수 있는 요약
```

필요하면 아래 공개용 문서도 만듭니다.

```text
공개용 프로필
- 블로그, 포트폴리오, 자기소개에 쓸 수 있는 공개용 버전
```

---

## 이 가이드가 아닌 것

```text
성격검사 아님
심리 진단 아님
자기소개서 자동 생성기 아님
팀원을 분류하거나 라벨링하는 도구 아님
직접 대화를 대체하는 도구 아님
최신 AI agent 구축 방법론 아님
사람을 자동 평가하는 시스템 아님
```
