## 여기서 볼 것

이 가이드를 텍스트로 진행할지, 음성으로 진행할지, ChatGPT Project나 파일 업로드를 쓸지 결정합니다.

---

## 먼저 결론

```text
텍스트만으로도 충분히 가능하다.
음성은 필수가 아니라 편한 입력 방식이다.
파일로 저장하는 것이 좋다.
가능하면 같은 채팅방이나 같은 Project에서 진행한다.
제품 기능은 계속 바뀌므로, 이 가이드는 특정 버튼보다 파일 구조와 검수 흐름을 기준으로 삼는다.
```

## 진행 방식 1: 직접 Markdown으로 작성

가장 안전하고 정돈된 방식입니다.

### 이 방식을 선택한 경우

1. [100문 100답 기록지](../templates/answer_log.md)를 복사합니다.
2. 100문 100답 기록으로 저장합니다.
3. 100문 100답을 보며 직접 답합니다.
4. 답변 파일을 ChatGPT에 첨부하거나 붙여넣습니다.

### 장점

```text
기록이 깔끔하다.
민감한 내용을 천천히 고를 수 있다.
나중에 파일로 관리하기 좋다.
```

### 단점

```text
쓰기 시작이 어렵다.
답변이 너무 정제되어 생생함이 줄 수 있다.
```

---

## 진행 방식 2: ChatGPT와 텍스트로 한 문제씩 진행

혼자 쓰기 어렵다면 이 방식을 추천합니다.

### 이 방식을 선택한 경우

1. ChatGPT에 새 채팅이나 Project를 엽니다.
2. [한 문제씩 진행하는 프롬프트](../prompts/02_one_question_mode.md)를 붙여넣습니다.
3. 질문 파일을 첨부하거나, “1번부터 시작해줘”라고 말합니다.
4. ChatGPT가 한 질문씩 묻고, 사용자는 답합니다.
5. 중간에 멈추고 싶으면 “여기까지 정리해줘”라고 말합니다.

### 장점

```text
부담이 적다.
한 번에 하나씩만 답하면 된다.
질문이 자연스럽게 이어진다.
```

### 단점

```text
채팅이 길어진다.
중간 정리를 자주 요청해야 한다.
```

## 진행 방식 3: ChatGPT와 음성으로 진행

음성은 머릿속에서 정리되지 않은 생각을 빠르게 꺼내는 데 좋습니다.

### Advanced Voice 또는 음성 대화란?

ChatGPT와 말로 대화하는 방식입니다. 텍스트로 치지 않고 말로 답변하고, ChatGPT의 응답도 음성으로 들을 수 있습니다. 여기서는 기능 이름보다 **말로 풀어내는 방식** 자체가 더 중요합니다.

### 왜 쓰나요?

여기서는 예쁜 문장을 쓰려는 게 아니라 원재료를 모읍니다. 말로 하면 이런 내용이 더 잘 나옵니다.

```text
아직 정리 안 된 생각
말하다가 떠오르는 기억
글로 쓰면 검열되는 감정
왜 그렇게 느꼈는지에 대한 즉흥적 설명
실제 대화의 뉘앙스
```

### 이 방식을 선택한 경우

1. ChatGPT에 새 채팅이나 Project를 엽니다.
2. 해당 단계의 프롬프트를 먼저 텍스트로 붙여넣습니다.
3. 필요한 파일을 첨부합니다.
4. 음성 대화를 켭니다.
5. ChatGPT가 한 질문씩 묻게 합니다.
6. 끝난 뒤 반드시 “오늘 대화를 Markdown 요약 파일로 정리해줘”라고 요청합니다.

### 장점

```text
생각을 빠르게 많이 꺼낼 수 있다.
글보다 덜 검열된 답변이 나온다.
긴 자기 설명을 덜 부담스럽게 할 수 있다.
감정과 뉘앙스가 더 잘 드러난다.
```

### 단점

```text
말이 산으로 갈 수 있다.
전사나 요약이 틀릴 수 있다.
민감한 내용을 더 쉽게 말해버릴 수 있다.
정확한 문장 인용에는 약하다.
대화 후 검수하지 않으면 기록 품질이 낮다.
```

## 어떤 모델을 써야 하나요?

강한 모델이 유리한 경우는 있습니다.

```text
자료가 많다.
긴 문서를 비교해야 한다.
모순을 찾아야 한다.
최종 보고서를 깊게 써야 한다.
```

하지만 꼭 특정 유료 모델이 필요하다는 뜻은 아닙니다. 모델이 약하거나 컨텍스트가 짧다면 파일을 쪼개고, 단계별로 진행하면 됩니다.

---

## 최신 기능을 꼭 따라가야 하나요?

아니요. ChatGPT Project, GPTs, 파일 업로드, 메모리, 음성 대화 같은 기능은 계속 바뀝니다. 이 가이드는 최신 기능을 다 쓰는 방법이 아니라, 제가 지금 하듯 필요한 기능만 붙이고 기록과 검수를 남기는 쪽에 가깝습니다.

```text
원자료를 모은다.
민감도와 사용 범위를 기록한다.
한 번에 만들지 않고 단계별로 생성한다.
제 말인지 GPT 해석인지 검수한다.
자주 쓸 맥락은 active_context로 줄인다.
```

새 기능이 생기면 이 흐름을 대체하기보다 보조 도구로 붙입니다.
