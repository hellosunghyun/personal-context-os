# Changelog

## v1.0.3

- 공개 리뷰 기반 개선 체크리스트를 추가.
- README에 대상 독자, 과할 수 있는 사용자, OS 용어 정의를 추가.
- START_HERE와 실행 흐름을 15분 시작판, 2시간 실사용판, 깊은 정식판으로 재구성.
- 100문 100답에 먼저 답할 필수 30문 경로를 추가.
- 15분/2시간 경로에서 전체 사용 설명서 없이 바로 미니 Active Context를 만들 수 있도록 프롬프트를 추가.
- 품질 테스트에 active_context before/after 비교와 1~5점 평가표를 추가.
- before/after 테스트가 같은 채팅 맥락에 섞이지 않도록 새 채팅 2개 비교 방식으로 수정.
- 보안 문서에 민감 자료를 실수로 올렸을 때의 삭제/대체/검수 루틴을 추가.
- source_register 기본 표를 줄이고, 원문 업로드 여부와 삭제/대체/반영 여부는 보안 추적 메모로 분리.
- 팀 공유 문서에 자발성, 회수 가능성, 인사평가 활용 금지 원칙을 추가.
- Active Context, 팀원용 1페이지, 나쁜 예시와 좋은 예시, 자료 목록 예시를 추가.
- 파트별 06A~06F 프롬프트를 추가하고 병합/검수/버전 분리 프롬프트를 보강.
- 리뷰용 단일 병합 파일은 커밋되지 않도록 제외.
- 링크 라벨과 일반 설명 문장에서 파일 경로가 먼저 보이던 부분을 자연어 문서명으로 정리.
- 100문 100답 기록지에 실제 Q1~Q100 질문 원문을 넣어 질문 파일을 따로 열지 않아도 답할 수 있게 정리.
- 목차를 첫 생성 흐름 중심으로 다시 정리해, 질문/프롬프트/템플릿/예시는 진행 중 참고 문서로 보이게 수정.
- 실제 운영 사례를 추가해, 가이드/결과물/원자료를 한 Project에 함께 넣는 방식과 그 보안 리스크를 설명.

## v1.0.2

- 라이선스를 MIT에서 `CC BY-NC-SA 4.0`으로 변경.
- 비상업적 학습, 스터디, 교육, 수정, 공유는 허용하고 상업적 사용은 별도 허가가 필요하도록 명확히 함.
- README에 라이선스 설명을 추가.
- 공개 문서에 맞게 반말 1인칭 표현을 `제가`, `저는`, `제` 기준으로 정리.
- 전체 파일을 다시 확인해 질문 수, 내부 링크, 첫 생성 흐름이 깨지지 않았는지 검증.

## v1.0.1

- 첫 생성 흐름과 완성 후 운영/확장 흐름을 분리.
- 최종 문서를 한 번에 생성하지 않고 파트별 초안 생성 후 합치는 방식으로 변경.
- 파트 초안 통합 프롬프트를 추가해 통합 단계를 명확히 함.
- 첫 생성 완료 페이지를 추가해 마지막 도착점을 명확히 하고, 9단계에서 100문 100답으로 돌아가던 순환 링크를 제거.
- 100문 100답 기록지를 Q1~Q100까지 바로 기록할 수 있게 확장.
- 자료 목록에 원문 위치, 업로드/보관 위치, 삭제/갱신 기준을 추가.
- 최종 생성/병합/검수 프롬프트에 근거 ID, 평가 불가, 추가 확인 필요 기준을 추가.
- Active Context에 사용 규칙, 사용 금지 정보, 최근성, 예시 요청을 추가.
- GPTs 확장 문서에 Instructions, Knowledge, Conversation starters, Capabilities 매핑을 추가.
- prompts, templates, examples, tests 사이의 뜬금없는 교차 navigation을 정리.

## v1.0.0

- 처음 보는 사람이 따라 할 수 있도록 전체 구조를 처음부터 재작성.
- 필수 흐름을 `준비 → 자료 수집 → 100문 100답 → 1차 인터뷰 → 추가 질문 → 2차 인터뷰 → 최종 문서 → 검수/분리 → 업데이트`로 정렬.
- 100문 100답을 쉬운 질문, 설명, 예시 중심으로 재작성.
- 텍스트 진행과 음성 진행을 각 단계에서 명확히 분리.
- 결과물을 Markdown 파일로 저장하는 방식을 기본으로 설계.
- 선택사항은 GPTs/챗봇, 팀 공유, 음성 사용 등으로 제한.
