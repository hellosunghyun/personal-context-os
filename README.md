# Personal Context OS Guide

AI가 저를 아는 척하지 않고, 제가 납득할 수 있는 답변을 만들게 하기 위한 공개용 워크북입니다.

이 레포로 하려는 일은 **가장 멋진 문장**을 만드는 게 아닙니다. 오히려 아래에 가깝습니다.

> 제가 어떤 맥락에서 어떤 답변을 납득하는 사람인지 정리하고, LLM이 그 맥락을 참고해 **제가 바로 쓸 수 있는 답변**을 더 자주 만들게 하는 것.

이 레포는 성격검사도, 자기소개서도, 프롬프트 모음집도 아닙니다.

**개인 맥락을 모으고, 정리하고, 검수하고, 계속 고쳐 쓰기 위한 실행 가이드**입니다.

여기서 말하는 OS는 소프트웨어 운영체제가 아닙니다. 개인 맥락을 수집하고, 압축하고, 검수하고, 업데이트하는 **운영 체계**라는 뜻입니다.

최신 AI 기능을 전부 따라가겠다는 문서는 아닙니다. 오히려 Memory, Projects, GPTs, 에이전트처럼 기능이 계속 바뀔수록 더 중요해지는 "내 맥락을 어떻게 정리하고 검수할 것인가"를 다룹니다.

제품 기능, 보안, 성격검사, LLM 개인화 관련 배경 자료는 [참고 자료](references.md)에 따로 모았습니다. 본문 안에서 출처가 필요한 정의나 주의사항은 각 문서 안에 짧은 출처 메모로 달아두었습니다.

---

## 왜 지금 필요한가

AI 도구는 빠르게 개인화되고 있습니다. Memory, Projects, GPTs, 파일 업로드, 에이전트 같은 기능은 점점 더 많은 맥락을 기억하고 활용합니다.

하지만 기능이 맥락을 대신 만들어주지는 않습니다.

AI가 기억해야 할 것과 기억하면 안 되는 것, 답변에 반영해야 할 기준과 오래되어 버려야 할 자기 설명, 제 말투를 살리는 수정과 과한 업그레이드는 사용자가 직접 구분해야 합니다.

이 가이드는 최신 AI 기능을 따라잡기 위한 문서가 아닙니다. 오히려 기능이 바뀌어도 계속 필요한 일을 다룹니다.

- 내가 어떤 답변을 납득하는지 정리하기
- AI가 추정으로 빈칸을 채우지 못하게 하기
- 좋은 답변과 제 답변을 구분하기
- 민감한 맥락과 공개 가능한 맥락을 나누기
- 오래된 자기 설명을 계속 업데이트하기

사람과 대화할 때도 우리는 상대에 맞춰 말합니다. 이 가이드는 AI와의 대화에서도 그런 조율이 가능하도록, 반복해서 쓸 수 있는 개인 맥락을 만드는 작업입니다.

더 자세한 배경은 [왜 지금 필요한가](docs/00_why_now.md)에서 볼 수 있습니다.

## 잘 맞는 사람

- LLM을 자주 쓰지만 답변이 제 맥락과 어긋난다고 느끼는 사람
- 자기소개, 회고, 피드백 소화, 커리어 문서를 자주 다루는 사람
- 매번 같은 배경 설명을 반복하지 않고, 개인 맥락을 파일로 관리하고 싶은 사람
- GPT 답변을 바로 쓰기보다 근거, 공개 범위, 저자성을 확인하고 싶은 사람

## 과할 수 있는 사람

- 빠른 프롬프트 팁 몇 개만 원하는 사람
- 자기 회고나 답변 기록을 길게 남기고 싶지 않은 사람
- 민감한 정보를 온라인 도구에 올리는 것이 불편한 사람
- 정량 평가나 자동화된 개인화 시스템을 기대하는 사람

이 가이드는 답변 품질 향상을 보장하지 않습니다. 대신 맥락을 남기고, 비교하고, 수정하는 과정을 만들어줍니다.

## 지금 당장 할 일

처음이라면 [시작하기](START_HERE.md)에서 아래 셋 중 하나를 고르면 됩니다.

```text
Level 1. 15분 시작판
- Q1~Q5만 답하고 미니 Active Context를 만든다.

Level 2. 2시간 실사용판
- 필수 30문과 1차 인터뷰로 Active Context를 만든 뒤 테스트한다.

Level 3. 깊은 정식판
- 100문 100답, 1차/2차 인터뷰, 파트별 초안, 병합/검수까지 진행한다.
```

전체 9단계는 [전체 실행 흐름](workflow/00_linear_flow.md)에 정리되어 있습니다. 처음부터 모두 끝낼 필요는 없습니다.

## 전체 흐름

처음에 만드는 것은 사용 설명서 초안입니다.

```text
1. 작업 공간 만들기
2. 선택 자료 모으기
3. 100문 100답 진행하기
4. 1차 인터뷰 진행하기
5. 추가 질문 생성하기
6. 2차 인터뷰 진행하기
7. 파트별 최종 문서 초안 만들기
8. 초안을 합치고 검수하기
9. 첫 생성 완료 상태 확인하기
```

업데이트 시스템, 팀 공유, GPTs/챗봇 확장은 **완성 뒤 필요할 때** 진행합니다.

각 단계는 [목차](SUMMARY.md)에 정리되어 있습니다.

## 음성으로 해야 하나요?

아닙니다.

- 텍스트로 혼자 작성해도 됩니다.
- ChatGPT와 텍스트 채팅으로 한 문제씩 진행해도 됩니다.
- ChatGPT 음성 대화로 편하게 말해도 됩니다.

음성은 필수가 아니라, **머릿속에서 아직 정리되지 않은 생각을 필터링 없이 말하기 쉬운 입력 방식**입니다. 자세한 설명은 [도구와 진행 방식](docs/02_tools_and_modes.md)과 [100문 100답 진행하기](workflow/03_answer_100_questions.md)를 보세요.

## 결과물

첫 생성 흐름을 끝까지 진행하면 아래 문서를 만들 수 있습니다.

```text
100문 100답 기록
1차 인터뷰 요약
추가 질문 목록
2차 인터뷰 요약
전체 사용 설명서
Active Context
팀원용 1페이지
```

필요하면 아래 문서를 추가로 만듭니다.

```text
공개용 프로필
업데이트 계획
답변 취향 로그
사건 기반 업데이트 로그
```

ChatGPT에게 결과를 받을 때는 채팅창에 긴 텍스트만 받지 말고, 가능하면 **Markdown 파일 형태로 생성해달라고 요청**하세요.
