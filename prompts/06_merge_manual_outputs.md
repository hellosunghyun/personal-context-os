## 언제 쓰면 좋나요?

`outputs/drafts/`에 파트별 초안이 만들어진 뒤, 최종 파일로 합칠 때 사용합니다.

## 넣을 자료

아래 초안 파일. 공개용 초안은 만들었을 때만 포함합니다.

```text
outputs/drafts/05a_identity_values.md
outputs/drafts/05b_work_communication.md
outputs/drafts/05c_llm_risks_growth.md
outputs/drafts/06a_active_context.md
outputs/drafts/07a_team_share_one_pager.md
outputs/drafts/08a_public_profile.md (공개용을 만들었을 때만)
```

## 나와야 할 결과

중복과 충돌을 정리한 최종 파일이 생성됩니다.

---

```text
아래 파일들은 한 사람의 개인 사용 설명서를 만들기 위해 파트별로 작성한 초안이다.

이번 작업에서는 새 내용을 더 쓰지 않는다. 초안을 정리해 최종 파일로 합친다.

중요:
- 초안에 없는 새 경험, 동기, 성향을 추가하지 않는다.
- 더 멋있는 문장으로 업그레이드하지 않는다.
- 중복은 줄이되 사용자의 말투와 판단 흐름은 보존한다.
- 서로 충돌하는 내용은 하나로 단정하지 말고 "충돌/추가 확인 필요"로 표시한다.
- 본인용, 팀원용, 공개용 공개 범위를 섞지 않는다.
- 민감한 내용은 팀원용/공개용으로 옮기지 않는다.
- 근거 ID가 있는 문장은 가능한 한 유지한다.
- 근거가 없는 중요한 해석은 확정하지 말고 "추가 확인 필요" 또는 "평가 불가"로 남긴다.

중복 정리 기준:
- 같은 의미의 문장이 단순 반복되면 합친다.
- 서로 다른 장에서 같은 패턴이 다른 맥락으로 드러나면 유지한다.
- 핵심 패턴은 한 번 요약하고, 각 장에서는 구체적 행동으로만 남긴다.
- 반복되는 표현을 줄이더라도 근거 ID와 공개 범위는 삭제하지 않는다.

입력 초안:
[여기에 outputs/drafts/ 파일들을 첨부하거나 붙여넣기]

출력 파일 1: outputs/05_full_manual.md
구성:
- 0. 이 문서의 사용법
- 1. 한 문장 설명
- 2. 현재의 자기 인식
- 3. 살아온 맥락과 반복 주제
- 4. 코어 드라이브와 가치관
- 5. 직관과 판단 방식
- 6. 일하는 방식
- 7. 커뮤니케이션 인터페이스
- 8. 에너지와 회복
- 9. 관계와 경계
- 10. LLM 답변 취향
- 11. 저와 일하는 법
- 12. 위기와 리스크
- 13. 성장 방향
- 14. Quick Command Palette
- 15. 추가 확인 필요

출력 파일 2: outputs/06_active_context.md
- 1~3페이지 안에 들어가도록 압축한다.
- 실제 LLM 대화에 붙일 수 있는 내용만 남긴다.
- 오래된 이야기, 장황한 배경, 본인용 감정 기록은 제외한다.
- 언제 붙이고 언제 붙이지 않을지 사용 규칙을 포함한다.
- 다시 확인할 날짜와 오래되어 의심되는 내용을 포함한다.

출력 파일 3: outputs/07_team_share_one_pager.md
- 팀원이나 협업자가 오해를 줄이고 협업 방식을 조정하는 데 참고할 수 있는 내용만 남긴다.
- 심리/건강/가족/회사 내부 정보는 제거하거나 행동 패턴으로 완화한다.

출력 파일 4: outputs/08_public_profile.md (공개용을 만들 때만)
- 공개 가능한 일하는 방식, 강점, 관심사, 협업 선호 중심으로 쓴다.
- 사적인 취약성, 회사/타인 정보, 진단처럼 보이는 표현은 제외한다.

마지막에 "합치기 검수 메모"를 작성한다.

합치기 검수 메모:
- 초안끼리 충돌한 내용:
- 추가 확인 필요로 남긴 내용:
- 평가 불가로 남긴 내용:
- 근거 ID가 약하거나 사라진 내용:
- 팀원용/공개용에서 제거한 민감 내용의 유형:
- 다음 단계에서 저자성 검수가 필요한 문장:
```
