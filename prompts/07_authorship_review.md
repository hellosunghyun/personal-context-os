# 프롬프트 07. 저자성 검수

[← 이전](06_merge_manual_outputs.md) | [목차](../SUMMARY.md) | [다음 →](08_split_versions.md)

## 언제 쓰면 좋나요?

최종 문서가 나온 직후 사용합니다. GPT 답변이 좋아 보여도 바로 쓰지 않기 위한 검수입니다.

## 넣을 자료

최종 문서 초안.

## 나와야 할 결과

제 말 같지 않은 부분, 과한 업그레이드, 근거 없는 해석이 표시됩니다.

---

```text
아래 문서는 한 사람의 개인 사용 설명서 초안이다.

검수에서는 문서를 더 멋지게 만들지 않는다.
이 문서가 실제로 사용자의 것이 맞는지 확인한다.

검수 기준:
1. 사용자가 말하지 않은 내용이 들어갔는가?
2. 심리나 동기를 단정한 문장이 있는가?
3. 너무 멋있게 업그레이드되어 사용자의 말 같지 않은 부분이 있는가?
4. 사용자가 요청할 때 얼핏 상상했을 방향과 다르게 흘러간 부분이 있는가?
5. 좋은 문장이지만 실제로 사용자가 쓰지 않을 표현이 있는가?
6. 본인용/팀원용/공개용 공개 범위가 섞였는가?
7. 민감하거나 위험한 정보가 남아 있는가?
8. 추가 확인 필요를 그럴듯하게 채운 부분이 있는가?
9. 근거 ID 없이 단정한 문장이 있는가?
10. 평가할 수 없는 내용을 평가한 것처럼 쓴 부분이 있는가?

출력 형식:

## 1. 바로 삭제해야 할 문장

## 2. 표현 수위를 낮춰야 할 문장

## 3. 근거가 부족한 해석

## 4. 너무 업그레이드된 문장과 대체 문장

## 5. 사용자의 말투에 더 가까운 수정본

## 6. 평가 불가 또는 보류해야 할 내용

## 7. 최종 채택 전 사용자가 직접 확인해야 할 질문

문서 초안:
[여기에 붙여넣기 또는 파일 첨부]
```
