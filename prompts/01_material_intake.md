# 프롬프트 01. 자료 분류

[← 이전](00_project_context.md) | [목차](../SUMMARY.md) | [다음 →](02_one_question_mode.md)

## 언제 쓰나요?

포트폴리오, 회고, 평가 요약, 검사 결과 같은 자료를 넣기 전에 민감도와 활용 방식을 분류할 때 사용합니다.

## 필요한 입력

자료 원문 또는 요약본. 민감한 원문은 먼저 가명화하세요.

## 기대 결과

`source_register.md`에 넣을 자료 목록과 사용 가능 범위가 나옵니다.

---

```text
아래 자료들은 한 사람이 자신의 사용 설명서를 만들기 위해 모은 원자료다.

이 작업의 목적은 자료를 바로 해석하는 것이 아니라, 먼저 안전하게 분류하는 것이다.

분류 기준:
1. 공개 가능
2. 개인적이지만 낮은 위험
3. 가명화 후 사용 가능
4. 본인용에만 사용
5. 사용하지 말 것

각 자료마다 아래 형식으로 정리해줘.

[자료명]
- 자료 ID:
- 자료 유형:
- 원문 위치:
- 업로드/보관 위치:
- 핵심 내용 요약:
- 민감도: 낮음 / 중간 / 높음 / 사용 금지
- 가명화 필요 여부:
- 최종 사용 설명서 활용 가능성:
- 본인용/팀원용/공개용 가능 범위:
- 삭제/갱신 기준:
- 주의할 점:
- 추가로 확인할 질문:

중요:
- 자료만 보고 사용자를 단정하지 않는다.
- 심리검사나 평가 결과를 정답처럼 쓰지 않는다.
- 회사 기밀, 타인 개인정보, 고객 정보가 보이면 사용하지 말고 제거/가명화를 제안한다.
- 원문을 업로드하지 않아도 되는 자료는 요약본만 쓰라고 제안한다.
- 나중에 삭제하거나 교체해야 할 자료는 삭제/갱신 기준을 적는다.
- 아직 최종 문서를 쓰지 않는다.

자료:
[여기에 자료를 붙여넣거나 파일로 첨부]
```
