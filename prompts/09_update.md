# 프롬프트 09. 완성 후 업데이트

[← 이전](08_split_versions.md) | [목차](../SUMMARY.md)

## 언제 쓰나요?

사용 설명서를 만든 뒤 월간 업데이트, 사건 기반 업데이트, active_context 압축에 사용합니다. 첫 생성 중에는 사용하지 않아도 됩니다.

## 필요한 입력

기존 `full_manual.md`, `active_context.md`, 최근 기록 또는 사건 로그.

## 기대 결과

변경분, archive 후보, active_context 수정안이 나옵니다.

---

```text
아래 자료는 한 사람의 개인 사용 설명서를 업데이트하기 위한 최근 기록이다.

목표는 전체를 다시 쓰는 것이 아니다.
목표는 바뀐 점만 찾고, 지금 LLM에게 필요한 핵심 맥락을 갱신하는 것이다.

입력:
- 기존 full_manual.md
- 기존 active_context.md
- 최근 기록 또는 event_update_log
- 좋았던/싫었던 GPT 답변 로그가 있다면 포함

업데이트 기준:
- 새로 알게 된 것
- 더 이상 현재와 맞지 않는 것
- 강해진 패턴
- 약해진 패턴
- 추가 확인 필요
- archive로 옮길 내용
- active_context에 반영할 내용

출력 형식:

## 1. 새로 추가할 내용

## 2. 수정할 내용

## 3. 더 이상 현재와 맞지 않는 내용

## 4. archive로 옮길 내용

## 5. 추가 확인 질문

## 6. active_context.md 수정안
1~3페이지 안에 들어가도록 압축한다.

## 7. answer_taste_log에 추가할 답변 취향

중요:
- 일시적 기분과 반복 패턴을 구분한다.
- 문서를 더 길게 만드는 것이 목표가 아니다.
- 지금 자주 쓸 맥락을 선별하는 것이 목표다.
```
