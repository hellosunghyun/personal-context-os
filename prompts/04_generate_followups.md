## 언제 쓰면 좋나요?

1차 인터뷰 이후, 최종 문서를 쓰기 전에 반드시 사용합니다.

## 넣을 자료

`01_answer_log.md`, `02_round1_summary.md`, 선택 자료 요약.

## 나와야 할 결과

`03_followup_questions.md`에 들어갈 추가 질문 목록이 나옵니다.

---

```text
아래 자료들은 한 사람의 개인 사용 설명서를 만들기 위한 원자료다.

최종 문서를 바로 쓰지 않는다.
최종 문서를 쓰기 전에 반드시 더 확인해야 할 질문을 만든다.

입력 자료:
- 100문 100답
- 1차 인터뷰 요약
- 선택 자료 요약

질문 생성 기준:
- 이미 답변된 내용을 반복하지 않는다.
- 실제 장면이 부족한 부분을 묻는다.
- 서로 충돌하는 내용을 확인한다.
- 공개 범위가 애매한 내용을 확인한다.
- 근거 ID가 없거나 약한 해석을 확인한다.
- 평가 불가로 남겨야 할 내용을 억지로 확정하지 않는다.
- 최종 사용 설명서의 어느 장에 필요한 질문인지 표시한다.
- 질문은 사용자를 분석하는 말투가 아니라, 자연스러운 인터뷰 질문이어야 한다.

출력 형식:

# 추가 인터뷰 질문

## 1. 반드시 물어봐야 할 질문 10개
각 질문마다:
- 질문:
- 왜 필요한가:
- 최종 문서의 어느 장에 반영되는가:
- 관련 근거 ID:
- 민감도: 낮음 / 중간 / 높음

## 2. 모순 해소 질문
각 항목:
- 충돌 지점:
- 확인 질문:

## 3. 실제 사례 보강 질문

## 4. 공개 범위 확인 질문

## 5. 답변 취향 확인 질문
이 사람이 어떤 답변을 자기 것처럼 느끼는지 더 알기 위한 질문.

## 6. 평가 불가로 남겨도 되는 항목
지금 질문해도 답이 안 나올 가능성이 높거나, 시간이 지나야 판단할 수 있는 항목.

금지:
- 질문에 대한 답을 대신 쓰지 않는다.
- 사용자의 성격이나 동기를 단정하지 않는다.
- 최종 문서를 아직 작성하지 않는다.
```
