# 프롬프트 06. 파트별 최종 문서 초안 생성

[← 이전](05_round2_interview.md) | [목차](../SUMMARY.md) | [다음 →](06_merge_manual_outputs.md)

## 언제 쓰면 좋나요?

모든 원자료를 모은 뒤, 최종 문서를 한 번에 만들지 않고 파트별 초안을 만들 때 사용합니다.

## 넣을 자료

`source_register.md`, `01_answer_log.md`, `02_round1_summary.md`, `03_followup_questions.md`, `04_round2_summary.md`, 선택 자료 요약.

## 나와야 할 결과

선택한 파트 하나에 해당하는 초안 파일이 생성됩니다.

---

## 사용 방법

아래 프롬프트를 복사한 뒤, `생성할 파트`와 `출력 파일명`을 하나만 지정합니다.

파트가 헷갈리면 아래 분리 프롬프트를 바로 사용해도 됩니다.

```text
06a_build_identity_values.md
06b_build_work_communication.md
06c_build_llm_risks_growth.md
06d_build_active_context.md
06e_build_team_share.md
06f_build_public_profile.md
```

권장 순서:

```text
파트 A: outputs/drafts/05a_identity_values.md
파트 B: outputs/drafts/05b_work_communication.md
파트 C: outputs/drafts/05c_llm_risks_growth.md
파트 D: outputs/drafts/06a_active_context.md
파트 E: outputs/drafts/07a_team_share_one_pager.md
파트 F: outputs/drafts/08a_public_profile.md (공개용이 필요할 때만)
```

한 번 실행할 때는 한 파트만 작성합니다.

---

```text
아래 자료들은 한 사람의 개인 사용 설명서를 만들기 위한 원자료다.

이번 작업에서는 최종 문서 전체를 한 번에 만들지 않는다.
지정된 파트 하나만 근거 기반으로 초안 작성한다.

생성할 파트:
[여기에 파트 A/B/C/D/E/F 중 하나만 입력]

출력 파일명:
[여기에 outputs/drafts/... 파일명 하나만 입력]

공통 작성 원칙:
1. 사용자가 직접 말한 내용과 해석을 분리한다.
2. 사용자가 말하지 않은 경험, 동기, 과거, 감정을 지어내지 않는다.
3. 확실한 내용, 가설, 추가 확인 필요를 구분한다.
4. 심리검사나 성향검사는 참고자료일 뿐 정답으로 쓰지 않는다.
5. 진단명이나 성격 낙인 대신 관찰 가능한 행동 패턴으로 쓴다.
6. 너무 멋있게 업그레이드하지 않는다.
7. 사용자의 원래 말투와 사고 순서를 가능한 한 보존한다.
8. 공개 범위를 본인용/팀원용/공개용으로 구분한다.
9. 회사 기밀, 타인 개인정보, 고객 정보, 민감한 원문은 포함하지 않는다.
10. 근거가 부족하면 문장을 만들지 말고 "추가 확인 필요"로 남긴다.
11. 중요한 문장에는 가능한 한 근거 ID를 붙인다.
12. 판단할 수 없는 항목은 억지로 평가하지 말고 "평가 불가"로 표시한다.

입력 자료:
[여기에 파일을 첨부하거나 요약을 붙여넣기]

근거 ID 규칙:
- source_register의 자료는 S001, S002처럼 쓴다.
- 100문 100답은 Q001, Q002처럼 쓴다.
- 1차 인터뷰는 R1, 2차 인터뷰는 R2로 쓴다.
- 근거가 섞이면 "근거: Q014, R2"처럼 표시한다.
- 근거가 기억나지 않으면 문장을 확정하지 말고 "추가 확인 필요"로 둔다.

파트별 작성 범위:

파트 A. 정체성, 맥락, 가치, 판단
출력 파일: outputs/drafts/05a_identity_values.md
포함할 장:
- 0. 이 문서의 사용법
- 1. 한 문장 설명
- 2. 현재의 자기 인식
- 3. 살아온 맥락과 반복 주제
- 4. 코어 드라이브와 가치관
- 5. 직관과 판단 방식

파트 B. 일, 소통, 에너지, 관계
출력 파일: outputs/drafts/05b_work_communication.md
포함할 장:
- 6. 일하는 방식
- 7. 커뮤니케이션 인터페이스
- 8. 에너지와 회복
- 9. 관계와 경계

파트 C. LLM 답변 취향, 협업법, 리스크, 성장
출력 파일: outputs/drafts/05c_llm_risks_growth.md
포함할 장:
- 10. LLM 답변 취향
- 11. 저와 일하는 법
- 12. 위기와 리스크
- 13. 성장 방향
- 14. Quick Command Palette
- 15. 추가 확인 필요

주의:
- 위기와 리스크는 진단하지 말고 행동 패턴으로만 쓴다.
- 성장 방향은 사용자가 직접 말한 목표나 선택 기준이 있을 때만 쓴다.
- 근거가 없으면 "아직 작성하지 않음" 또는 "추가 확인 필요"로 남긴다.

파트 D. Active Context
출력 파일: outputs/drafts/06a_active_context.md
목표:
LLM에게 자주 첨부할 1~3페이지 핵심 맥락 파일.
포함할 내용:
- 사용 규칙: 언제 붙이고, 언제 붙이지 않을지
- 이 사람에게 좋은 답변의 기준
- 피해야 할 답변
- 선호하는 문체와 구조
- 현재 중요한 우선순위
- 판단 기준
- 답변 전에 물어볼 질문
- 공개 주의 정보
- 사용 금지 정보
- 최근성: 기준일, 다시 확인할 날짜, 오래되어 의심되는 내용
- 예시 요청
- 마지막 업데이트 날짜

파트 E. Team Share One Pager
출력 파일: outputs/drafts/07a_team_share_one_pager.md
목표:
팀원이나 협업자에게 공유 가능한 1페이지 요약.
포함:
- 저와 일할 때 좋은 방식
- 업무 요청법
- 피드백 방식
- 갈등 시 확인 질문
- 하지 말아야 할 것
제외:
- 건강/심리 세부 정보
- 사적인 가족/관계 내용
- 회사 내부 민감 정보

파트 F. Public Profile
출력 파일: outputs/drafts/08a_public_profile.md
목표:
블로그, 포트폴리오, 자기소개에 활용 가능한 공개용 버전.
포함:
- 일하는 방식
- 강점
- 관심사
- 협업 선호
- 어떤 문제를 잘 푸는지
제외:
- 민감한 자기노출
- 타인/회사 관련 정보
- 진단처럼 보이는 표현

주의:
- 공개용 문서가 필요 없으면 이 파트는 만들지 않는다.

출력 규칙:
- 요청한 파트 하나만 작성한다.
- 다른 파트의 내용을 미리 쓰지 않는다.
- 최종 파일로 합치지 않는다.
- 마지막에 아래 체크리스트와 근거 추적 메모를 붙인다.

검수 체크리스트:
- 사용자가 말하지 않은 내용을 만들지 않았는가?
- 너무 멋있게 업그레이드하지 않았는가?
- 사용자의 말투와 결을 보존했는가?
- 공개 범위가 표시되었는가?
- 추가 확인 필요를 숨기지 않았는가?

근거 추적 메모:
- 이 파트의 핵심 근거:
- 근거가 약한 문장:
- 평가 불가로 남긴 항목:
- 다음 단계에서 반드시 사용자에게 확인할 질문:
```
