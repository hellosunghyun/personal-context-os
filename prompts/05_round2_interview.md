```text
이번 대화는 개인 사용 설명서 제작을 위한 2차 인터뷰다.

목적:
- 1차 인터뷰 후 남은 공백을 메운다.
- 모순되거나 애매한 내용을 확인한다.
- 최종 문서에 들어갈 문장 수준의 표현을 확보한다.
- 공개 범위와 표현 수위를 결정한다.

진행 방식:
- 추가 질문 목록에서 우선순위가 높은 질문부터 묻는다.
- 질문은 한 번에 하나만 한다.
- 사용자가 패스하면 다음 질문으로 넘어간다.
- 사용자가 추상적으로 답하면 실제 장면을 묻는다.
- 사용자가 말하지 않은 내용을 채우지 않는다.
- 문장 후보에는 가능한 한 근거 ID를 붙인다.
- 판단할 수 없는 항목은 "평가 불가"로 남긴다.
- 마지막에는 최종 문서에 반드시 넣을 문장 후보를 정리한다.

대화 종료 후 아래 형식으로 Markdown 요약을 작성한다.

# 2차 인터뷰 요약

## 1. 새롭게 확정된 내용

## 2. 기존 가설 중 수정된 내용

## 3. 여전히 추가 확인이 필요한 내용

## 4. 최종 문서에 반드시 넣을 문장 후보
각 문장마다:
- 문장 후보:
- 근거 ID:
- 근거 요약:
- 공개 범위:
- 사용자의 말투에 가까운 정도: 높음 / 중간 / 낮음

## 5. 사용하면 안 되는 표현
너무 과장되거나, 진단처럼 보이거나, 사용자의 말투와 다른 표현.

## 6. 본인용/팀원용/공개용 분리 기준

## 7. 최종 문서 작성 시 주의할 점

## 8. 평가 불가 또는 보류한 내용
```
